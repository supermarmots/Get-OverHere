# 약속 추천 날짜 기획

## 목적

수집된 참여자 가능 일정 데이터를 바탕으로 주최자에게 가장 적합한 날짜 후보를 최대 3개까지 보여준다. 별도 응답 수집 페이지는 1차 범위에서 제외한다.

## 사용자 흐름

1. 주최자가 대시보드에서 약속 상세 화면으로 들어간다.
2. 상세 화면에서 `추천 날짜` 영역을 확인한다.
3. 참여자가 2명 이상이면 시스템은 availability를 실시간 집계해 득표가 높은 날짜 3개를 표시한다.
4. 후보가 없으면 아직 추천할 수 없다는 빈 상태를 보여준다.

## 추천 기준

- 입력 데이터: `meetings/{meetingId}/participants/*`의 `availability`
- 집계 단위: 날짜
- 점수: 해당 날짜를 선택한 참여자 수
- 표시 조건: 참여자 2명 이상
- 정렬:
  - 1순위: 참여자 수 높은 순
  - 2순위: 날짜 빠른 순
- 최대 표시 개수: 3개

## 화면 구성

### 상세 화면 추천 영역

- 제목: `추천 날짜`
- 후보 항목:
  - 날짜와 요일
  - 가능한 참여자 수
- 빈 상태:
  - `아직 추천할 수 있는 후보가 없습니다.`

## Firestore 작업

- `meetings/{meetingId}/participants` 서브컬렉션 실시간 구독
- 클라이언트에서 날짜별 집계
- 1차 구현에서는 `meetings/{meetingId}.recommendation`에 저장하지 않고 화면 계산만 수행한다.

## 구현 메모

- 같은 참여자가 같은 날짜에 여러 시간대를 입력해도 날짜 득표는 1표로 계산한다.
- 동점일 때는 빠른 날짜를 먼저 표시한다.
- 추천 날짜는 참여자 변경 사항을 Firestore snapshot으로 실시간 반영한다.
