# 로그인 후 랜딩 페이지 기획

## 목적

로그인한 사용자가 가장 먼저 보는 화면은 `메인 대시보드`다. 이 화면은 약속을 새로 만들거나, 초대받은 약속에 참여하거나, 진행 중인 약속 상태를 빠르게 확인하는 업무 시작점이어야 한다.

## README 기준 기능 연결

- 회원 시스템: 로그인한 사용자 이름/닉네임을 상단에 표시한다.
- 약속 조율 주최자: `약속 만들기`를 가장 중요한 액션으로 제공한다.
- 약속 참여자: `초대 링크로 참여` 진입점을 제공한다.
- 실시간 모니터링: 진행 중 약속의 후보 날짜/참여 현황을 요약한다.
- 최종 선정: 확정 대기 또는 확정 완료 상태를 구분해서 보여준다.

## 화면 구성

### 상단 영역

- 서비스명 `Get Over Here`
- 사용자 인사말: `{닉네임}님, 약속을 조율해 볼까요?`
- 보조 액션: 로그아웃, 프로필

### 주요 액션 영역

- `약속 만들기`: 캘린더 기반 후보 날짜/시간 선택 화면으로 이동
- `초대 링크로 참여`: 공유받은 URL 또는 초대 코드 입력

### 내 약속 요약

탭 또는 섹션으로 구분한다.

- `주최 중`: 내가 만든 약속 방
- `참여 중`: 내가 투표해야 하거나 투표한 약속
- `확정됨`: 최종 일정이 정해진 약속

각 약속 항목은 카드 UI보다 목록형 행으로 표시한다.

- 약속 제목
- 상태: 투표 중, 확정 대기, 확정됨
- 후보 날짜 수
- 참여자 수
- 마지막 업데이트
- 진입 버튼: 보기

## 모바일 UX

- 첫 화면에서 인사말, `약속 만들기`, `초대 링크로 참여`가 바로 보여야 한다.
- 목록은 한 열로 표시하고, 각 행의 주요 정보는 2줄 이내로 제한한다.
- CTA 버튼은 터치하기 쉽게 전체 너비를 사용한다.

## 데이터 초안

Firestore 예상 구조:

- `users/{uid}`: 사용자 프로필
- `meetings/{meetingId}`: 약속 방 기본 정보
- `meetings/{meetingId}/participants/{uid}`: 참여자 응답

대시보드에서는 우선 다음 쿼리가 필요하다.

- 내가 `hostId`인 약속
- participants에 내 uid가 있는 약속
- 확정 상태인 내 관련 약속
