# 로그인 후 대시보드 기획

## 목적

로그인한 사용자가 가장 먼저 보는 화면은 `/dashboard`다. 대시보드는 새 약속을 만들거나 초대 링크로 참여하고, 내가 관련된 약속의 진행 상태를 빠르게 확인하는 업무 시작점이다.

## 현재 구현 상태

- 로그인 성공 후 `/dashboard`로 이동한다.
- 비로그인 사용자가 보호 route에 접근하면 `/login`으로 이동하고, 로그인 후 원래 route로 돌아간다.
- 이미 로그인한 사용자가 `/login`, `/signup`에 접근하면 `/dashboard` 또는 저장된 redirect 목적지로 이동한다.
- 대시보드는 `participantIds`에 현재 uid가 포함된 약속을 Firestore에서 조회한다.
- 조회한 약속을 page state에서 다음 3개 섹션으로 분류한다.
  - 내가 조율 중: `hostId === uid`, `status !== confirmed`
  - 참여 중: `hostId !== uid`, `status !== confirmed`
  - 확정 완료: `status === confirmed`
- soft delete된 약속은 service에서 제외한다.
- 대시보드 상단에는 약속 현황 3개 지표를 1행 3열로 보여준다.
- 주요 액션은 `새 약속`, `초대 참여` 2개 버튼을 1행 2열로 보여준다.
- 초대 참여 버튼은 링크 입력 dialog를 열고, `/meetings/{meetingId}/join` 경로만 허용한다.

## 화면 구성

### 상단 영역

- eyebrow: `오늘의 약속 조율`
- 인사말: `{닉네임}님, 시간 되시나요?`
- 설명: `링크로 가능 시간을 모으고, 겹치는 날짜를 확인한 뒤 확정하세요.`
- 보조 액션: `로그아웃`

프로필 버튼은 실제 기능이 없으므로 제거했다.

### 약속 현황

1행 3열 레이아웃을 유지한다.

- 내가 조율 중
- 참여 중
- 확정 완료

각 항목은 숫자와 라벨만 간결하게 표시한다.

### 주요 액션

1행 2열 레이아웃을 유지한다.

- `새 약속`: `/meetings/new`로 이동
- `초대 참여`: 초대 링크 입력 dialog 열기

버튼 안의 한 줄 설명은 제거했다.

### 약속 목록

- `MeetingListSection` 3개를 순서대로 표시한다.
- 약속 항목은 `MeetingCard`로 표시한다.
- 카드 전체가 상세 화면 진입 액션이다.
- 확정 완료 섹션에서 주최자는 `재개최하기` 액션으로 status를 `collecting`으로 되돌릴 수 있다.

### 빈 상태

관련 약속이 하나도 없으면 시작 안내를 보여준다.

```text
아직 약속이 없습니다
새 약속을 만들거나 받은 초대 링크로 참여하면 이곳에서 진행 상황을 볼 수 있습니다.
```

## 데이터 기준

- 조회: `meetings` collection where `participantIds array-contains uid`
- 필터 제외: `status === deleted`
- 정렬: `createdAt` 최신 순
