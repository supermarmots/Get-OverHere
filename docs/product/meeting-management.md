# 약속 조회/수정/삭제 기획

## 목적

주최자와 참여자가 약속 상세 화면에서 약속 정보, 참여자, 추천 날짜, 초대 링크, 내 가능 일정을 확인하고 필요한 액션을 수행할 수 있게 한다.

## 현재 사용자 흐름

1. 대시보드 약속 카드를 누른다.
2. `/meetings/{meetingId}` 상세 화면으로 이동한다.
3. 약속 제목, 설명, 상태, 희망 날짜(월), 확정 일정, 참여자, 초대 링크, 추천 날짜, 내 가능 일정을 확인한다.
4. 주최자는 진행 중 약속에서 `확정하기`, `수정`, `약속 삭제`를 사용할 수 있다.
5. 주최자는 확정된 약속에서 `수정` 대신 `재개최하기`를 사용할 수 있다.
6. 참여자는 `수정`, `참여 취소`를 사용할 수 있다.
7. 주최자의 `수정`은 `/meetings/{meetingId}/edit`로 이동한다.
8. 참여자의 `수정`은 `/meetings/{meetingId}/join`으로 이동해 기존 응답을 수정한다.
9. 삭제/참여 취소는 확인 dialog 후 처리한다.

## 상세 화면 구성

- 뒤로가기 버튼
- 약속 제목
- 상태
- 설명
- 희망 날짜(월)
- 확정 일정
- 참여자 수와 참여자 이름 목록
- 초대 링크와 복사 버튼
- 추천 날짜 영역
- 내 가능 일정
- 하단 액션

## 액션 권한

### 주최자

- `확정하기`: 추천 날짜 중 하나와 선택 입력한 시작/종료 시간을 저장하고 `collecting` 상태를 `confirmed`로 변경
- `수정`: 약속 기본 정보와 주최자 가능 일정 수정
- `재개최하기`: 확정된 약속의 확정 결과를 초기화하고 `collecting` 상태로 되돌림
- `약속 삭제`: meeting 문서와 participant 서브컬렉션 문서를 삭제

### 참여자

- `수정`: join flow로 이동해 내 availability 수정
- `참여 취소`: participant 문서를 삭제하고 parent meeting의 `participantIds`에서 uid 제거

주최자의 participant 문서는 삭제할 수 없다.

## 수정 화면

1차 구현은 단계형 생성 UI를 재사용하지 않고 한 화면 편집으로 유지한다.

- 약속 제목
- 설명
- 희망 날짜(월)
- 내 가능 날짜/시간

희망 날짜(월)을 바꾸면 기존 가능 일정은 해당 월과 맞지 않을 수 있으므로 초기화한다.

## 삭제 정책

현재 약속 삭제는 hard delete다. 주최자가 삭제를 확정하면 `meetings/{meetingId}/participants/*` 문서를 먼저 batch에 담고, 마지막에 `meetings/{meetingId}` 문서를 함께 삭제한다.

삭제된 약속은 Firestore에 남지 않으므로 대시보드와 상세 조회에서 별도 `deleted` 상태를 필터링하지 않는다.

주의:

- 클라이언트 batch 삭제이므로 참여자 수가 Firestore batch 제한을 넘는 규모가 되면 별도 서버 함수나 soft delete 전환이 필요하다.
- 삭제 후 초대 링크나 상세 URL로 접근하면 약속 없음 상태로 처리한다.
