# 약속 조회/수정/삭제 기획

## 목적

주최자가 생성한 약속을 대시보드에서 확인하고, 필요하면 내용을 수정하거나 삭제할 수 있게 한다. 이 기능은 약속 생성 이후 관리 흐름의 기본 CRUD다.

## 사용자 흐름

1. 대시보드 `주최 중` 카드에서 약속을 누른다.
2. `/meetings/{meetingId}` 상세 화면으로 이동한다.
3. 약속 제목, 설명, 조율 월, 주최자의 가능 일정을 확인한다.
4. `수정`을 누르면 수정 화면으로 이동한다.
5. `삭제`를 누르면 확인 모달을 띄운다.
6. 삭제를 확정하면 Firestore에서 약속을 삭제하고 대시보드로 돌아간다.

## 화면 구성

### 상세 화면

- 약속 제목
- 설명
- 조율 월
- 상태
- 참여자 수
- 내 가능 일정
- 액션:
  - `수정`
  - `삭제`
  - `초대 링크 보기`
  - `대시보드로 이동`

### 수정 화면

1차 구현에서는 기존 단계형 생성 UI를 재사용하지 않는다. 오버엔지니어링을 피하기 위해 한 화면에서 다음 항목만 수정한다.

- 약속 제목
- 설명
- 조율 월
- 내 가능 날짜/시간

단, 수정 폼이 복잡해지면 이후 단계형 수정 플로우로 분리한다.

### 삭제 확인

- 불투명한 배경 위 중앙 모달
- 문구: `이 약속을 삭제할까요?`
- 보조 문구: `삭제하면 참여자 응답도 함께 사용할 수 없습니다.`
- 액션:
  - `취소`
  - `삭제`

## Firestore 작업

### Read

- `meetings/{meetingId}` 읽기
- `meetings/{meetingId}/participants/{uid}` 읽기

### Update

- `meetings/{meetingId}` 업데이트
- 주최자의 `participants/{uid}.availability` 업데이트

### Delete

Firestore 클라이언트에서 서브컬렉션 전체 삭제는 안전하게 처리하기 어렵다. 1차 구현은 실제 delete 대신 soft delete를 사용한다.

```js
{
  status: 'deleted',
  deletedAt: serverTimestamp(),
  updatedAt: serverTimestamp()
}
```

대시보드와 상세 화면에서는 `status === 'deleted'` 약속을 숨긴다.

## 권한

- 주최자만 상세 조회/수정/삭제 가능
- 참여자는 이후 참여 페이지에서 별도 권한 설계

## 1차 개발 범위

- [x] `/meetings/{meetingId}` 상세 경로 추가
- [x] `/meetings/{meetingId}/edit` 수정 경로 추가
- [x] 대시보드 카드 클릭 시 상세 화면 이동
- [x] 상세 조회 서비스 추가
- [x] 수정 서비스 추가
- [x] soft delete 서비스 추가
- [x] 삭제 확인 모달 추가
- [x] 대시보드에서 deleted 약속 제외

## 구현 메모

- 대시보드 약속 카드는 단일 주요 액션만 가지므로 별도 `보기` 버튼 없이 카드 전체 선택으로 상세 화면에 진입한다.
- 카드 진행 상태는 제목 반대편에 배치해 목록을 빠르게 훑을 수 있게 한다.
- 수정 화면에서 조율 월을 변경하면 기존 가능 일정은 월이 달라질 수 있어 초기화한다.

## 범위 제외

- 참여자용 상세 화면
- 서브컬렉션 hard delete
- 수정 화면 단계형 UX
- 실시간 추천 결과
- 참여자 응답 수정/삭제
