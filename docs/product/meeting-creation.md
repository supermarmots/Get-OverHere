# 약속 만들기 페이지 기획

## 목적

주최자가 약속 방을 만들고 본인이 가능한 날짜와 시간대를 입력하는 단계형 플로우다. 이 서비스의 핵심은 주최자와 참여자가 각자 가능한 일정을 표시하면, 다수결 원칙으로 가장 적합한 시간대를 추천하는 것이다.

주최자는 모든 후보 일정을 완성해서 제시하는 사람이 아니라, 해당 달에서 본인이 가능한 날짜/시간을 먼저 입력하고 참여자를 초대한다.

한 페이지에서 모든 정보를 한꺼번에 입력받지 않는다. 토스 같은 모바일 금융 앱처럼 한 단계에 하나의 결정만 요청해 사용자의 부담을 줄인다.

## 사용자 흐름

1. 대시보드에서 `약속 만들기`를 누른다.
2. 단계 1에서 약속 이름을 입력한다.
3. 단계 2에서 약속 설명을 선택적으로 입력한다.
4. 단계 3에서 조율할 월을 선택한다.
5. 단계 4에서 가능한 날짜를 선택한다.
6. 단계 5에서 선택한 날짜별 가능한 시간대를 입력한다.
7. 단계 6에서 입력 내용을 확인하고 생성한다.
8. 생성 성공 후 초대 링크를 공유한다.
9. 참여자도 같은 방식으로 가능한 날짜와 시간대를 표시한다.
10. 서비스는 참여자 응답을 집계해 가장 많은 사람이 가능한 시간대를 추천한다.

## 입력 항목

- 약속 제목: 필수
- 약속 설명: 선택
- 조율 월: 필수
- 내 가능 일정: 최소 1개 필수
  - 날짜
  - 시작 시간: 선택
  - 종료 시간: 선택

## 1차 UI 원칙

- 다크 테마 유지
- 모바일 우선 단일 컬럼 폼
- 카드형 UI 남발 금지
- `fieldset`, `label`, `input`, `textarea`, `button` 등 시맨틱 폼 태그 사용
- 가능한 날짜는 월 캘린더 UI로 선택한다.
- 가능한 시간 입력은 선택 사항이다.
- 월 단위 조율이라는 개념은 유지한다.
- 한 단계에 하나의 주된 입력만 둔다.
- 하단 고정 CTA 영역을 사용한다.
- 상단 왼쪽에는 언제든 약속 생성을 취소할 수 있는 버튼을 둔다.
- 취소 버튼 클릭 시 확인 모달을 띄워 실수로 나가는 상황을 막는다.
- 진행률은 상단 오른쪽에 표시한다.
- 이전/다음 이동을 명확하게 제공한다.
- 단계 진행률은 짧게 표시한다. 예: `3 / 6`

## 단계별 화면 구성

### 1. 약속 이름

- 질문: `어떤 약속인가요?`
- 입력: 약속 제목
- CTA: `다음`

### 2. 약속 설명

- 질문: `참여자에게 알려줄 내용이 있나요? (선택사항)`
- 입력: 설명
- 선택 입력이며 건너뛰기 가능
- CTA: `다음`

### 3. 조율 월

- 질문: `어느 달 안에서 만날까요?`
- 입력: 연도 이동과 월 선택이 가능한 Windows 달력형 UI
- CTA: `다음`

### 4. 가능한 날짜

- 질문: `가능한 날짜를 골라주세요`
- 입력: 해당 월의 캘린더에서 날짜 선택
- 여러 날짜를 토글 방식으로 선택/해제한다.
- 선택한 월을 캘린더 상단에 표시한다.
- 날짜는 실제 요일에 맞춰 배치한다.
- 기본 날짜 버튼 배경은 제거하고, 선택된 날짜만 강조한다.
- CTA: `다음`

### 5. 가능한 시간

- 질문: `선택한 날짜에 가능한 시간을 알려주세요`
- 입력: 날짜별 시작/종료 시간
- 선택 사항이며, 비워두면 해당 날짜는 `시간 미정`으로 저장한다.
- CTA: `다음`

### 6. 확인

- 입력한 약속 이름, 설명, 조율 월, 가능 일정을 요약한다.
- 가능 일정에는 날짜와 요일을 함께 표시한다. 예: `5월 12일 (화)`
- CTA: `약속 만들기`

## Firestore 데이터 초안

`meetings/{meetingId}`

```js
{
  title: string,
  description: string,
  hostId: string,
  targetMonth: 'YYYY-MM',
  status: 'collecting',
  participantIds: [hostUid],
  recommendation: null,
  createdAt: serverTimestamp(),
  updatedAt: serverTimestamp()
}
```

`meetings/{meetingId}/participants/{uid}`

```js
{
  uid: string,
  displayName: string,
  role: 'host' | 'participant',
  availability: [
    {
      id: string,
      date: 'YYYY-MM-DD',
      startTime: 'HH:mm' | '',
      endTime: 'HH:mm' | ''
    }
  ],
  createdAt: serverTimestamp(),
  updatedAt: serverTimestamp()
}
```

추천 결과는 응답 집계 단계에서 계산한다. 1차 저장 시점에는 `recommendation: null`로 둔다.

향후 추천 데이터 예시:

```js
{
  date: 'YYYY-MM-DD',
  startTime: 'HH:mm',
  endTime: 'HH:mm',
  availableCount: number,
  participantCount: number
}
```

## 검증 규칙

- 제목은 공백 제외 1자 이상
- 조율 월은 필수
- 내 가능 일정은 최소 1개
- 각 가능 일정은 날짜가 필요
- 가능 일정 날짜는 조율 월에 포함되어야 함
- 시간을 입력하는 경우 시작 시간과 종료 시간을 모두 입력해야 함
- 시간을 입력하는 경우 종료 시간은 시작 시간보다 늦어야 함
- 빈 일정 행은 제출 전에 에러 처리

## 1차 개발 범위

- `/meetings/new` 라우트 추가
- `src/features/meetings` 생성
- `CreateMeetingPage` 추가
- 단계 상태와 현재 단계 인덱스는 페이지 로컬 state 사용
- 폼 상태는 페이지 로컬 state 사용
- 단계별 컴포넌트는 `features/meetings/components`에 분리
- 검증 함수는 `features/meetings/lib`에 분리
- Firestore 생성 서비스는 `features/meetings/services`에 분리
- 주최자의 가능 일정을 `participants/{uid}`에 `role: 'host'`로 저장
- 생성 성공 시 우선 대시보드로 돌아가거나 성공 메시지 표시
- 대시보드 `약속 만들기` 버튼을 `/meetings/new`로 연결
